Terminal Reference

Git & GitHub
Repository Guide

A complete, step-by-step guide to creating and pushing a repository to GitHub — entirely from the command line.

⚡

All Commands — Quick Copy

# 1. Check if a git repo already exists
git status

# 2. Initialize a new git repository
git init

# 3. Stage all files for tracking
git add .

# 4. Commit staged files with a message
git commit -m "Initial commit"

# 5. Authenticate with GitHub (one-time)
gh auth login

# 6. Create the repo on GitHub, link & push
gh repo create your-repo-name --public --source=. --push

Step-by-Step Breakdown

Every command explained — what it does and what each part means.

1

Check Current Status

See if Git is already tracking this folder
PowerShell 
PS C:\project> git status

This is your starting point. Before doing anything, check whether Git is already initialized in this folder.

PartMeaning
git Calls the Git program — the version control system installed on your computer.
status A sub-command that shows the current state: which files are tracked, changed, or staged.
💡 If you see "fatal: not a git repository", it means Git is not set up yet — proceed to Step 2. If you see file listings, Git is already active.
2

Initialize the Repository

Turn your folder into a Git-tracked project
PowerShell 
PS C:\project> git init
Initialized empty Git repository in C:/project/.git/

This creates a hidden .git folder inside your project. That folder is where Git stores all version history.

PartMeaning
git Calls the Git program.
init Short for initialize. Creates a brand-new, empty Git repository in the current folder.
📁 The .git folder is hidden. It contains all commits, branches, and history. Never delete it unless you want to remove Git tracking entirely.
3

Stage All Files

Tell Git which files to include in the next save
PowerShell 
PS C:\project> git add .

Staging is like placing items on a table before packing them into a box. You choose what goes in.

PartMeaning
git Calls the Git program.
add The staging command — marks files to be included in the next commit (save point).
. The dot means "everything" — all files and folders in the current directory and subdirectories.
💡 To stage only one specific file instead of all, use: git add filename.html
4

Commit (Save a Snapshot)

Permanently save staged files into Git history
PowerShell 
PS C:\project> git commit -m "Initial commit"
[master (root-commit) 259a496] Initial commit
 1 file changed, 3 insertions(+)

A commit is a permanent snapshot of your project at this moment. Think of it as a save point in a video game — you can always come back to it.

PartMeaning
git Calls the Git program.
commit Saves all staged files as a new snapshot in the project history.
-m Short for --message. Lets you add a description right in the command line.
"Initial commit" Your message — a short note describing what this commit contains. Use quotes around it. You can write anything here, e.g. "Added login page".
⚠️ Without the -m flag, Git will open a text editor for you to type the message. Using -m is faster and simpler.
5

Authenticate with GitHub

One-time login so your terminal can talk to GitHub
PowerShell 
PS C:\project> gh auth login

This command opens an interactive login flow. GitHub needs to know who you are before letting you create repositories.

PartMeaning
gh Calls the GitHub CLI (Command Line Interface) — a separate tool from Git, made specifically for GitHub.
auth The authentication module — handles login and tokens.
login Starts the login process. You'll be asked to choose GitHub.com, HTTPS, and browser login.
🔑 You only need to do this once per machine. After logging in, your credentials are saved locally.
6

Create Repo, Link & Push

Create the GitHub repository and upload your code in one command
PowerShell 
PS C:\project> gh repo create your-repo-name --public --source=. --push
✓ Created repository username/your-repo-name on github.com
  https://github.com/username/your-repo-name
✓ Added remote https://github.com/username/your-repo-name.git
✓ Pushed commits to https://github.com/username/your-repo-name.git

This is the final and most powerful command. It does three things at once: creates the repo on GitHub, links your local folder to it, and uploads your code.

PartMeaning
gh Calls the GitHub CLI tool.
repo The repository module — handles creating, cloning, and managing repos.
create The action: create a new repository on GitHub.
your-repo-name The name you want for your repo on GitHub. Replace this with your desired name (e.g., www1, my-project).
--public Makes the repo visible to everyone. Use --private instead if you want only you to see it.
--source=. --source tells GitHub CLI where your local code lives. The dot (.) means "this current folder."
--push Automatically uploads (pushes) your committed files to the new GitHub repo right after creating it.
🎯 Without --source=. and --push, the repo is created on GitHub but stays empty — your local code won't be uploaded.
✓

Verify — You're Done!

Confirm everything is connected and pushed
PowerShell 
PS C:\project> git remote -v
origin  https://github.com/username/your-repo-name.git (fetch)
origin  https://github.com/username/your-repo-name.git (push)

This shows the remote connections — where Git will push and pull code from.

PartMeaning
git Calls the Git program.
remote Manages connections to remote repositories (like GitHub).
-v Short for --verbose. Shows the full URL of each remote connection.
origin The default name Git gives to your primary remote. It points to your GitHub repo URL.

🔄 Daily Workflow — After Initial Setup

Once your repo is created, these 3 commands are all you need to save and push changes:

PowerShell — Daily Commands 
PS C:\project> git add .
PS C:\project> git commit -m "describe your changes"
PS C:\project> git push
🚀 git push — uploads your new commits to GitHub. No flags needed after the initial setup because Git already remembers where to push.
